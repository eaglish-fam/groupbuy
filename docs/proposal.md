## 對齊基礎

這個站的本質：

- **IP-driven 策展型聚合站**（鷹式一家頻道 → eaglish.store）
- 連結點出去到**外部廠商團購表單**（不是站內結帳）
- 卡片有不同類型，**SEO 戰略要依類型分層**

| 卡片類型 | 該不該做 SEO | 原因 |
|---|---|---|
| 常駐團購（long） | ✅ 值得 | Evergreen，可 indexable |
| 書籍（康先生的書） | ✅ 強烈值得 | 長尾、IP 強連結 |
| 教育公益（edu） | ✅ 值得 | 長期內容、低競爭關鍵字 |
| 折扣碼（coupon） | ⚠️ 看狀況 | 變動快，常駐品牌可以 |
| **短期團購（short）** | ❌ **不該做** | 兩週後就死頁，反而傷站體質 |
| 即將開團（upcoming） | ❌ 不該做 | 同理 |

**站內留存比 SEO 拉新更高 ROI**，因為主流量從 YT/IG 來，不是 Google。

## A. SEO（只押 evergreen + IP，不押短期）

| 提案 | 槓桿 | 工程 | 適用範圍 / 為什麼 |
|---|---|---|---|
| **策展主題頁系統**（自動從 Sheet tag 生成「鷹式推薦清單」） | ⭐⭐⭐⭐⭐ | 中 | 全範圍 / 這才是 IP-driven 站該有的 SEO 武器，「2026 母嬰必買」「Hiram 私心愛用」長尾關鍵字 evergreen |
| **常駐團購 + 書籍**獨立 URL + Schema.org Product/Book | ⭐⭐⭐⭐ | 中 | long / book / edu 限定 / 這些不會死頁，值得 indexable |
| **FAQPage schema**（Sheet 既有 QA 欄位） | ⭐⭐⭐ | 低 | long / book 限定 / 搜尋結果展開常見問題 |
| **動態 sitemap.xml**（只列 evergreen 卡片） | ⭐⭐⭐ | 低 | long / book / edu / 排除 short / upcoming |
| **動態 OG 圖片**（每張 evergreen 卡片自己的 1200×630） | ⭐⭐⭐ | 中 | long / book / edu / FB/LINE 分享預覽會帶商品圖 |

## B. 轉換率（短期/長期都吃）

| 提案 | 槓桿 | 工程 | 為什麼 |
|---|---|---|---|
| **「Hiram 一句話推薦」**（Sheet 加一欄個人短評） | ⭐⭐⭐⭐⭐ | 極低 | 護城河就是 IP 信任，這個廠商抄不走，每張卡都該有 |
| **緊迫感 UI**：截止 24h 紅色脈衝、卡片即時倒數 | ⭐⭐⭐⭐⭐ | 低 | 短期團購本質是 FOMO，現在卡片本身完全沒急迫感，只在行事曆顯示 |
| **多通路最低價標記**（Sheet retailers 加價格欄） | ⭐⭐⭐⭐ | 中 | 信任 + 立刻可決策 |
| **affiliate UTM 自動注入** | ⭐⭐⭐ | 極低 | 從各通路後台精準歸因哪張卡賺最多 |

## C. 站內留存（流量從 YT/IG 來，留人比拉新重要）

| 提案 | 槓桿 | 工程 | 為什麼 |
|---|---|---|---|
| **Web Push 開團通知**（已有 PWA + SW，加訂閱即可） | ⭐⭐⭐⭐⭐ | 中 | 「即將開團」「最後 6 小時」自動 ping，不靠 SEO 也回流，是團媽社群最強留存武器 |
| **收藏雲端化 + Email 通知**（取代純 localStorage） | ⭐⭐⭐⭐ | 中 | 換手機就消失，現在收藏功能是半廢的 |
| **「✋ 我有興趣」即將開團登記** | ⭐⭐⭐ | 中 | upcoming 卡片可登記 → 開團主動 push |

## D. 短期團購的「防禦動作」（避免拖累整站 SEO）

| 提案 | 槓桿 | 工程 | 為什麼 |
|---|---|---|---|
| **過期卡片自動 `noindex`**（meta tag 條件式注入） | ⭐⭐⭐⭐ | 低 | 防止累積 soft 404 拖累 ranking |
| **過期卡片自動沉底 + 灰色化 UI** | ⭐⭐⭐ | 低 | 部分已有，可加強視覺 |
| **短期團購 deep link 不丟 sitemap** | ⭐⭐⭐ | 極低 | 跟 A 類「動態 sitemap」一起做 |
