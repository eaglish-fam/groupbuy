Layer 6 · Execution daemon（trading-agent · Python 24/7）

核心模組（src/trading_agent/）

entry point · _active_session() 路由 morning / evening / idle

quote_server.py
本地 quote 快取
bridge_client.py
讀取 trading-shared 內 plan JSON

tick loop 行為（每 30s）

  1. 判斷 active session（morning / evening / idle）
  2. 讀對應 plan JSON（morning: trade-plans/，evening: trade-plans-evening/）
  3. pull quote（Shioaji sim 或 live）→ 寫 runtime
  4. 比對 trigger_price / SL / TP1 / TP2
  5. 觸發 → 下單 + telegram + 寫 trade-logs/
  6. breaker 檢查（虧損 > NT$3000 / 連虧 3 次）
  7. 寫 heartbeat → Dashboard 讀
paper-trading preview 階段：Shioaji 用 simulation layer，plan 寫但不下 live 單；上線只需切 EXCHANGE env。

二、每日執行時間軸（TPE 時區）

07:00
morning-plan v1 — launchd trading-plan → wrapper → morning-plan.sh → claude 產 TXF/MXF/TMF plan → trade-plans/YYYY-MM-DD.json
08:45
台指期開盤 — daemon 切 morning session，開始監看 trigger_price
09:00–13:30
mid-session 仲裁（每 30min） — launchd trading-midsession → HOLD / TIGHTEN / FLAT / REVERSE
13:45
台指期收盤 — daemon 最後一 tick，所有部位 EOD 平倉（若未觸 TP/SL）
14:00–20:00
idle window — daemon 每 tick 跑 _refresh_quotes_idle()（六檔輪巡，Dashboard 報價不凍）
20:00
evening-plan v1 — US/Oil 前瞻，用 1H/4H/1D 結構產 SXF/TGF/BRF plan
21:30
evening-plan v2 — 15m/30m reality check（美股即將開盤）
21:45
evening-plan v3 — US 第一根 15m bar 完，final tune
22:15–03:45
evening mid-session（每 30min） — 12 slots，22:45 catch 週三 EIA 22:30
22:30 (週三)
EIA 週報 — event_driven_bias 觸發 BRF flat（v1 一律不交易）
05:00
夜盤 valid_until — evening plan 失效
收盤後
review-day.sh — 寫 trade-reviews/YYYY-MM-DD.md

三、Strategy Library（Phase 0 · 2026-04-20 新加）

當前狀態： pure markdown 資產，prompts 還沒接上來選。morning-plan.md / evening-plan.md 仍照舊產 plan。Phase 1 實作時主 prompt Step 5 會改成「從此 library 挑一個策略套用」，並在 plan JSON 加 strategy_used 欄。

策略檔 Status 適用 day-type 核心邏輯
trend_day_breakout.md active trend_day_up/down · breakout_day · gap_and_go Breakout（突破 + 量） 或 Pullback（趨勢回測 EMA）· 結構 SL 1–1.5 ATR · R:R 1:2 起、目標 1:3+
range_bound_fade.md reference-only range_day · reversal_day · failed_breakout · gap_and_fill 朋友的第二隻腳法 — 第一次觸及只標記、第二次回測看抗跌/抗漲訊號才進場 · SL 緊貼區間外 0.3–0.5 ATR · TP 對側邊緣 · 天然 1:3+。待 live log 校準
event_driven_bias.md active any · 盤中有 EIA / FOMC / earnings / CPI 凌駕規則 — 事件期間強制 flat · 事件後分 4 路（趨勢型→breakout、震盪→fade、消息反向→fade 反向、反應不明→flat）
defensive_flat.md active tight_consolidation · inside_day · unclear 正式的「今日不交易」決定 · 結構性理由一旦觸發整時段不解除 · 健康占比 20–40% 交易日

五、關鍵設計不變量（踩過的坑）

  • Bridge 在 ~/dev/，不在 iCloud — launchd 無法 exec iCloud 路徑的 .sh（Operation not permitted）；2026-04-20 把 bridge 全搬出 iCloud。
  • trading-shared 留在 iCloud — 那是資料層，跨設備同步是對的；只有「被 exec 的 script」需逃離。
  • claude -p 必須白名單 MCP tools — launchd 環境無法回應 permission prompt，用 --allowed-tools "Read,mcp__tradingview" 明確允許，避開 bypassPermissions（被 sandbox 擋）。
  • daemon plan 路徑每 tick 重算 — 不要在 daemon 啟動時算一次；長 run daemon 會跨日，否則隔天讀錯 plan。
  • idle window 仍需 _refresh_quotes_idle() — 不跑 tick 會讓 Dashboard 的 last_quote_ts 凍結；順便早期驗證 SXF/TGF/BRF contract 解析。
  • evening JSON extractor 要 robust — claude 夜盤常 prepend prose，用 python regex 抓 ```json fence 取代 sed strip first/last line。
  • Strategy library Phase 0 零侵入 — markdown only，不改現行 prompt；確保 library 本身穩定後才讓 Phase 1 的 prompt 去選。
